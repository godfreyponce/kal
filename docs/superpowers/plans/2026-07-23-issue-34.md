**Goal:** Restyle the Chat screen (`/chat`) into the #28 silent-menu language — cream canvas, chocolate/caramel/red-brown palette, uppercase sans micro-labels — matching `design/chat-silent-menu-combined.html` (round-3 pick: V1 + three meta-strip tweaks), touching **only** styling.

The Chat CSS section of `app/globals.css` is rewritten block-by-block from the current cool-grey `--ink`/`--surface`/`--border`/`--faint` palette to the shared `--sm-*` silent-menu palette (with three promoted tokens + one new one). `app/chat/chat.tsx` gets **class-name and inline-style changes only** (add the shared `.anim` entrance class, split the two cost/token spans so they can be coloured independently). **Tech Stack:** Next.js (App Router, the repo's forked version), React client component, vanilla CSS with `@theme` tokens in `app/globals.css`.

## Open decisions (owner: confirm at the plan gate)

These three calls are the ones most likely to be wrong. Everything else follows the mockup mechanically.

1. **The `"Ephemeral, not saved"` kicker — keep it as two spans (no comma), or inject the comma?**
   The mockup and the Done-when text write it with a comma: *"Ephemeral, not saved"*. But the live markup is two separate spans (`<span>Ephemeral</span><span>not saved</span>`) rendered with a gap, and the Constraints say **no copy changes**. Injecting a literal comma changes the rendered characters — a copy change. **Recommendation:** keep the two-span structure, restyle it to a single caramel uppercase line (`EPHEMERAL   NOT SAVED`, 6px gap). If the owner would rather have the comma, that's a one-character markup change to make instead. **This plan assumes: keep two spans, no comma.**

2. **Promote the three repeated literals to `--sm-*` tokens and refactor the existing usages, or add tokens used only by Chat?**
   `#f1e7d3` (1 existing use), `#b3a390` (5 existing uses), and `#e2d9c9` (1 existing use) are already hard-coded literals elsewhere and the Chat spec reuses all three. To honour "no duplicated hex" the way #32 did (it promoted `--gro-*` → `--sm-*`), **Recommendation:** promote all three to `--sm-band-deep` / `--sm-muted` / `--sm-border`, and refactor the ~7 pre-existing literals to the tokens in the same pass (pure same-value swaps, zero visual change). If the owner wants a tighter diff, descope Step 2 (the pre-existing-literal refactor): Chat still uses the new tokens (introducing no new duplication), and the old literals stay as they are. **This plan assumes: promote + refactor.**

3. **Empty-state (`.chat-empty`, "Ask Kal") styling — not shown in the mockup.**
   The ticket says it should "adopt the language too; keep its copy." **Recommendation:** "ASK KAL" becomes uppercase sans-800 chocolate (matching every other silent-menu title); the sub-line stays normal-case, coloured caramel. Copy unchanged. **This plan assumes that treatment.** If the owner prefers the sub-line also uppercased, say so.
