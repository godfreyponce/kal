# Issue #27 — Em-dash / middle-dot copy sweep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove every em dash (—) and middle dot (·) from user-visible UI copy in `app/`, replacing each so the copy still reads naturally.

**Architecture:** Pure string edits — a fixed table of exact before/after replacements across 12 files. No markup, logic, or CSS changes. Verified by a grep that must come back empty for non-comment hits, plus the existing suite.

**Tech Stack:** Next.js 16 App Router, TypeScript, vitest.

## Global Constraints

- Owner copy rule: no em dashes (—) or middle dots (·) in UI copy; use commas or plain spacing.
- Copy-only change: no markup, logic, or CSS changes (issue #27 Constraints).
- Chat/model-generated text is out of scope — `lib/system-prompt.ts` and `lib/tools.ts` prompt text stays untouched, as do all code comments and CSS comments.
- Repo is PUBLIC — no env values, credentials, or owner personal data in any committed file.
- Single-task plan → work stays **uncommitted in the working tree** (AGENTS.md); the owner reads the diff at gate 2.
- `npm test` needs `DATABASE_URL` (hits live Neon) and runs sequentially; `npx tsc --noEmit` must stay clean.

## Scope notes (decided at planning time)

- **No new tests.** This is a copy-only sweep; no existing test asserts any of these strings (verified by grep at planning time). Success criterion = the grep check in Step 2 + clean `tsc` + green suite.
- The grep at planning time found exactly the 29 string occurrences tabled below. All other hits in `app/` (including every hit in `app/globals.css`) are code comments — explicitly out of scope per the ticket's Done-when.
- **Replacement style:** rule-literal — commas (or a plain rewording where a comma reads badly, e.g. the lookup placeholder). No em dashes swapped for periods/colons/hyphens, so the diff stays predictable.
- The two `"—"` null-value glyphs in `profile-section.tsx` (lines 164/167) ARE user-visible strings, so the Done-when's grep criterion covers them. Replaced with `"n/a"` (lowercase, matching the app's lowercase copy style).
