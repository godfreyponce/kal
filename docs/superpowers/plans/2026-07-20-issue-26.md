## Global Constraints

- Motion per `docs/design/motion-and-feel.md`: press response starts <~50ms; scale never below 0.9 (press token 0.97, enter range 0.94–1.0 — the hold here uses 0.94, the deep end of the sanctioned range); release settles with slight overshoot, never a hard stop; `prefers-reduced-motion` drops ALL transform/spring (non-negotiable). No new dependency for springs.
- **Scope decision (settled in planning):** bubble press applies to the 7 weekly-strip cells only. The calendar-sheet chips (`.cal-cell`/`.cal-chip` in `adherence-calendar.tsx:328-332`) are non-interactive display `<div>`s — nothing to press, no press feel.
- Must not fight the two existing gestures on the same strip: tap opens the day-detail modal (`weekly-adherence.tsx` `onOpen`), swipe-up on the strip arms the calendar-sheet pull (`adherence-calendar.tsx:206-273`, listener on `stripRef` = the `.adh-body` div). Part 1 adds NO JS and NO listeners, so it cannot intercept either; the only interplay is visual (see Task 1 contingency).
- UI copy: no em dashes, no middle dots (owner rule). The only copy in play is the word `history` and aria-label `Show adherence history` — keep both exactly.
- `.cell.ahead` buttons are `disabled` — they must not depress (`:active` doesn't fire on disabled buttons; keep the `:not(:disabled)` guard anyway for clarity).
- Repo is PUBLIC: no env values or personal data in code/comments.
- Turbopack serves stale CSS after `globals.css` edits — `rm -rf .next`, restart dev, hard-refresh before judging any visual change (STATE.md gotcha).
- `npm test` needs `DATABASE_URL` (hits live Neon); `npx tsc --noEmit` must stay clean.

In `app/globals.css`, add a `transition` line to the existing `.cell` rule (line ~1719) and a new `:active` rule directly after `.cell:disabled` (line ~1725):

```css
.cell {
  flex: 1; display: flex; flex-direction: column; align-items: center; gap: 5px;
  position: relative; cursor: pointer;
  border: 0; background: none; font: inherit; padding: 4px 0 0; margin: 0;

  transition: transform 0.30s cubic-bezier(0.34, 1.56, 0.64, 1); /* release: spring settle w/ overshoot */

  transform: scale(0.94) translateY(1px);
  transition: transform 0.45s cubic-bezier(0.16, 0.9, 0.3, 1);
}
```

**Phone-tuning contingency (do NOT build pre-emptively).** Known data point from planning: the static mockup's press preview was dead on the owner's phone until a `touchstart` shim was added — iOS Safari ignores `:active` on touch when a page has no touch listeners. The real app has plenty (React events + the strip's pointer listeners) and the groceries `:active` press already passes on this phone, so `:active` is expected to fire in-app; the residual risks are feel, not activation. Two known iOS risks, both only judgeable on the phone at gate 2: (a) iOS Safari can delay `:active` beyond the <50ms response target; (b) during a swipe-up pull the finger stays down, so the pressed cell may stay compressed while the sheet rises (possibly fine, possibly noisy). If the owner's phone pass fails on either, replace the `:active` selector with a JS-driven class, keeping the identical CSS curves: in `weekly-adherence.tsx`'s `Cell`, add

```tsx
const [pressing, setPressing] = useState(false);
const y0 = useRef(0);
// on the <button>: (skip for the disabled "ahead" branch)
onPointerDown={(e) => { y0.current = e.clientY; setPressing(true); }}
onPointerMove={(e) => { if (pressing && Math.abs(e.clientY - y0.current) > 8) setPressing(false); }}
onPointerUp={() => setPressing(false)}
onPointerCancel={() => setPressing(false)}
className={pressing ? "cell pressing" : "cell"}   // today/ahead keep their extra class
```

and change the CSS selector `.cell:active:not(:disabled)` → `.cell.pressing` (the 8px move threshold releases the bubble the moment a pull or scroll starts, which also solves risk b).
