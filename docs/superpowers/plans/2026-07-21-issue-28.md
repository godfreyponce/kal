# Silent-Menu Design Language Rollout (issue #28) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Roll the Groceries silent-menu design language (issue #1) across Today, Login, Chat, and Plan via one owner-gated mockup round per screen, ending with one filed build ticket per screen. This is an umbrella ticket: **#28 itself ships no source code** — the code lands under the per-screen tickets it spawns.

**Architecture:** Owner decisions (2026-07-21, /plan-ticket session): all four screens are in scope; the cream-vs-white canvas question is decided from the first mockup round, not up front; #28 is umbrella + first mockups, one build ticket per screen spawned as picks land. Each round = one `design/*-silent-menu-variants.html` file with 3 variants → owner pick → commit the file + `gh issue create` the screen's build ticket. Rounds run Today → Login → Chat → Plan.

**Tech Stack:** Static HTML mockup files in `design/` (self-contained, Google-fonts only). `gh` CLI for ticket filing. No app code.

## Global Constraints

- Repo is PUBLIC: no env values, credentials, or owner personal data in mockups or issues.
- No em dashes (—) or middle-dot (·) separators in any in-phone mockup copy (owner rule, #27). Page-chrome captions are exempt but keep them clean anyway.
- Standing design rule: 3 HTML variants per screen, owner picks before any build ticket is worked.
- Every per-screen build ticket uses the agent-task template sections (What / Done when / Constraints / Probably touches / Provenance) and is NOT labeled `ready-for-agent` — the owner green-lights labels himself.
- Mockups must include `@media (prefers-reduced-motion: reduce)` disabling the enter stagger, matching prod behavior.

## The style ingredients (source of truth)

From `design/groceries-rework-combined.html` and its landed CSS (`app/globals.css` 1311–1490, scoped to `.groceries`):

| Ingredient | Value |
|---|---|
| Canvas | white `#fff` (vs app cream `--canvas: #f7f6f3`) |
| Chocolate ink | `#3f2a1a` (`--gro-choc`) |
| Pill brown | `#48291a` (`--gro-pill`) |
| Caramel accent | `#a97a24` (`--gro-caramel`) |
| Red-brown | `#ac2318` (`--gro-red`), active-tab underline 2.5px |
| Cream band | `#f6efe2`, radius 10, padding 9px 12px, 10px/700/.14em uppercase |
| Names | uppercase, sans 800, letter-spacing .01em |
| Pills | 8.5px/700/.1em uppercase, radius 999, padding 4px 10px; kcal = pill brown, protein = caramel |
| Enter stagger | `translateY(16px) scale(.97)` → none, .34s `cubic-bezier(.32,.72,.33,1)` backwards, ~30ms per item capped at 300ms |
| Imagery | free-floating white product shots, `mix-blend-mode: multiply` (Groceries-specific; mostly N/A elsewhere) |

Current app baseline (what the variants restyle): one `app/globals.css` (1868 lines) with per-screen sections; `:root` tokens at lines 9–33; cream on `body` line 36; PWA `themeColor: "#F7F6F3"` in `app/layout.tsx:27`. Groceries is the only screen overriding the canvas (`body:has(main.groceries){background:#fff}`, globals.css 1321).

## Sequencing notes

- **Order rationale:** Today first because it is the hub Groceries links back to, it settles the canvas question, and its build ticket owns the shared-token promotion (`--gro-*` → `--sm-*`) everything else consumes. Login second as the trivial round that cements the vocabulary. Chat third. Plan last — biggest screen, and it inherits both the shared modal styles the Today ticket touches and the outcomes of #30/#31.
- **#29 (groceries edit form revamp)** explicitly waits on this rollout's style decisions — its variants should be drawn only after round 1's pick.
- **Build tickets are independent of later rounds:** the Today build can be green-lit and built while rounds 2–4 are still drawing variants; rounds only gate their own screen's ticket.
