# Login Silent-Menu Restyle (#33) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restyle the Login screen to match `design/login-silent-menu-combined.html` (the #28 round-2 V1 pick plus the owner's two tweaks), using the shared `--sm-*` tokens from #32.

**Architecture:** Pure styling + two copy changes. `app/login/page.tsx` loses its card wrapper and moves the error line inside the input's column; the login block of `app/globals.css` (lines 1290–1344) is rewritten in the silent-menu language. The load stagger reuses the shared `.anim` class + `@keyframes enter` that #32 already added at `app/globals.css:72-80` (reduced-motion off-switch included) — no new keyframes.

**Tech Stack:** Next.js App Router (client component), vanilla CSS in `app/globals.css`. No new dependencies.

## Global Constraints

- **Off-limits:** `lib/session.ts` and the `/api/auth/login` handler — login flow and session logic untouched. The page's `submit()` function, state hooks, and fetch call do not change; only JSX classes/structure and copy.
- **Copy:** kicker reads exactly `You shall not pass`; error line reads exactly `Incorrect password` (no trailing period). Never em dashes (—) or middle dots (·) in copy.
- **Tokens:** all five silent-menu palette hexes come from the shared tokens (`--sm-choc`, `--sm-pill`, `--sm-caramel`, `--sm-red`, `--sm-band`) — no duplicated palette hex. Auxiliary mockup literals (`#e2d9c9` input border, `#b3a390` placeholder, `rgba(172, 35, 24, 0.12)` focus ring, `#fff`) stay literal — that matches how #32's Today CSS already writes them (`app/globals.css:290,304`).
- **No new dependencies.** Repo is PUBLIC — no env values or personal data in any file.
- **Single-task plan** → per AGENTS.md it stays **uncommitted in the working tree**; the owner reads the diff at gate 2. Do not commit.

## Decisions locked in (and why)

1. **The button gets a new `.login-btn` class instead of restyling `.btn-dark`.** `.btn-dark` is shared by weigh-in, Plan editors, Chat, and Groceries (`grep -rn btn-dark app/`); the pill needs different values for nearly every property, so a scoped override pile would be messier than a dedicated class. The existing `.login .btn-dark {}` override rule is deleted. `:active` scale and `:disabled` opacity are carried over so press feel and the disabled-when-empty behavior match the rest of the app.
2. **The input's error-state class (`err`) is dropped.** The spec draws no red border on the input in the error state — the red-brown *focus ring* plus the red-brown error line carry the signal. `className={error ? "err" : ""}` leaves the JSX and `.login input.err {}` leaves the CSS.
3. **Stagger delays are inline styles** (`style={{ animationDelay: "0.05s" }}`), exactly the #32 Today pattern (`app/page.tsx:87,114`).
4. **`.login-inner` max-width is 300px** (mockup's `.inner`), down from the card's 320px.
5. **`busy ? "…" : "Enter"` stays.** "…" is an ellipsis, not a middle dot; the mockup doesn't model the busy state, so current behavior is kept.
